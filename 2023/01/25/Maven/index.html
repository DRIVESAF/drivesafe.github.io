Maven简介

Maven是什么

  • Maven本质是一个项目管理工具，将项目开发和管理过程抽象成一个项目对象模型（POM）

  • POM：项目对象模型

Maven的作用

  • 项目构建：提供标准的、跨平台的项目构建方式

  • 依赖管理：方便快捷的项目依赖的资源（jar包），避免资源间的版本冲突问题

  • 统一开发结构：提供标准的、统一的项目结构

Maven基础概念

仓库

仓库：用于存储资源包，包含各种jar包

中央仓库：Maven团队维护的，储存所有开源资源的仓库

私服仓库：部门、公司维护的，从中央获取资源的仓库

仓库分类：

  • 远程仓库：

计算机→本地仓库→私服仓库→中央仓库

计算机→本地仓库→中央仓库

  • 本地仓库：

计算机→本地仓库

私服的作用：

  • 保存具有版权的资源，包含购买或自主研发的jar（央仓库中的jar都是开源的，不能存储具有版权的资源）
  • 一定范围内共享资源，仅对内部开放，不对外共享
  • 解决中央仓库访问速度慢的问题

坐标

坐标：Maven中的坐标用于描述仓库中资源的位置

Maven坐标的主要组成：

（组织ID）groupld：定义当前Maven项目隶属组织名称（通常是域名反写）

（项目ID）artifactld：定义当前Maven项目名称（通常是项目模块名称）

（版本号）version：定义当前版号

（packaging：定义该项目的打包方式）

Maven坐标的作用：

使用唯一标识，唯一性定位资源位置，通过该标识可以将资源的识别与下载工作交由机器完成

本地仓库配置（资源下载到的位置）

Maven启动后，会自动保存下载的资源到本地仓库

  • 默认位置

    ${user.home}/.m2/repository

  • 自定义位置

    D:\maven\repository

远程仓库配置（资源从哪来）

在setting文件中配置镜像仓库

全局setting与用户setting的区别

  • 全局setting定义了当前计算机中Maven的公共配置
  • 用户setting定义了当前用户的配置

生命周期

项目构建生命周期

Maven对项目构建的生命周期划分为3套

一、clean：清理工作

  • pre-clean 执行一些需要在clean之前完成的工作
  • clean 一处所有上一次构建生成的文件
  • post-clean 执行一些需要在clean之后立刻完成的工作

二、default：核心工作（如：编译、测试、打包、部署…）

compile 编译项目的源代码

test 用单元测试框架运行测试（比如Juint）

package 将编译后的代码打包成可分发格式的文件（如jar、war…）

install 安装项目到本地仓库，可以作为依赖

……

三、site：产生报告，发布站点…

  • pre-site 执行一些需要在生成站点文档之前完成的工作
  • site 生成项目的站点文档
  • post-site 执行需要在生成站点文档之后完成的工作，并且为部署做准备
  • site-deploy 将生成的站点文档部署到特定的服务器上

插件

插件与生命周期内的阶段绑定，在执行到对应生命周期时执行对应的功能

默认maven在各个生命周期上绑定有预设的功能

通过插件可以自定义其他功能
