Linux命令

  • cd :改变目录

  • cd ..：回到上一级

  • pwd：显示当前所在的目录路径

  • clear：清屏

  • ls：列出当前文件夹所有文件（类似dir）

  • touch：在当前目录新建一个文件

  • rm：移除一个文件（删除）

  • mkdir：创建一个目录（文件夹）（定义文件类型如：aaa.md）

  • rm -r：删除目录

  • mv：移动文件（例：mv aaa.md bbb）（aaa为移动目标，bbb为放置目标）

  • history：查看命令历史

  • help：帮助

  • exit：退出

  • #：注释

  • reset：初始化终端（重新加载Git Bash）

Git工作原理

Git四个工作区域

  • 工作区：平时存放代码的地方

  • 暂存区（其实是个文件）：暂存改动文件

  • 本地仓库：存放提交的所有版本的数据

  • 远程仓库：托管代码

Git工作流程

正向操作：本地文件（git add.）👉暂存区（git commit）👉本地Git仓库（git push）👉远程仓库

反向操作：远程仓库（git pull）👉本地仓库（git reset）👉暂存区（git checkout）👉本地

Git配置

配置用户信息
(注意在第一个引号前需要打空格)

git config –global user.name ‘your_name’

git config –global user.email ‘your_email@domain.com‘

config的三个作用

git config –local #只对某个仓库有效

git config –global #对当前用户所有仓库有效

git config –system #对系统所有登录的用户有效

（显示现有配置：加 –list，如：git config –list –local）

Git项目搭建

本地搭建仓库（在当前目录创建一个Git代码库）

git init（创建出的.git文件是个隐藏文件）

克隆远程目录（镜像仓库）

git clone + 仓库的地址

两种场景：

1.把已有的项目代码纳入Git管理

cd 项目代码所在的文件夹

git init

2.新建的项目直接用Git管理

cd 某个文件夹

git init your_project #在当前路径下创建和项目名称同名的文件夹

cd your_project #进入刚刚创建的文件夹

Git操作

文件的4种状态

使用 git status 命令查看文件状态

  • untracked：未跟踪。文件未加入至git库中，不参与版本控制。
  • unmodify：文件已入库，未修改。
  • modified：文件已修改，未进行其他操作（如未入库）。
  • staged：文件暂存。

忽略文件

*.txt 忽略所有.txt结尾的文件

！lib.txt 但lib.txt除外

/temp 仅忽略此目录下的文件，子目录temp中的不忽略

build/ 忽略目录build下的所有文件

doc/*.txt 会忽略某个文件夹下的某个东西

重命名文件

git readme redeme.md

git readme（要变更的文件） redeme.md（重命名设置）

git log查看版本历史

git log –oneline：简洁查看历史列表

git log -n4 –oneline：查看最近四个历史并生成历史

git log –all –graph：图形化查看分支演化

git常用命令

git init #初始化本地库

git status #查看本地库状态

git add . #文件名

git commit -m”日志信息” 文件名 #提交到本地库

git reflog #查看历史记录

git reset –hard 版本号 #版本穿梭

git推送命令
git status #查看本地库状态
git add . #添加到暂存区
git commit -m”日志信息” 文件名 #提交到本地库
git push -u origin 分支名 #推送至远程
